## Description

Set up a local, file-backed task tracker that AI tools (Claude Code) can manage and a human can review. Goal: lightweight Jira-alternative — markdown source of truth, HTML viewer, no server, no cloud.

## Work log

- 2026-05-06: agreed on the design — one markdown file per task in `tasks/`, `TASKS.md` index regenerated by a build script, self-contained `tasks.html` viewer.
- 2026-05-06: wrote `build_tasks_html.py` (parses frontmatter, regenerates `TASKS.md` and `tasks.html`).
- 2026-05-06: added `/task-start` and `/task-done` slash commands.
- 2026-05-06: wired `PostToolUse` hook in `.claude/settings.json` so the HTML refreshes whenever a task file is edited.

## Summary

Tracker is live. Schema: `id`, `title`, `status` (active/backlog/done), `area`, `priority`, `started`, `finished` in frontmatter; body has `## Description`, `## Work log`, `## Summary`. Use `/task-start <title>` to open, `/task-done <id>` to close. Open `tasks/tasks.html` in a browser to review.
